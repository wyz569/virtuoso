Product Version IC23.1, September 2023

B

SDR Options Form

Use the SDR Options form to specify options that usually do not change for simulation driven interactive routing.

Field Description

Automatic pin connections

This section lets you specify the options for automatically connecting pins.

Targets Type

Selects one of the following types for the creation of automatic connections to unrouted pins and existing pathSegs.

  • Inst pins only: Creates automatic connections to pins that are not connected to any routing object.
  • PathSegs and pins: Creates automatic connections to any of the following:
    • pins that are not connected to any routing object
    • segments connected and aligned with pins
    • trunk segments indirectly connecting (through segments or vias) multiple pins

Environment variable: weAutoTwigTargetsType

Via Mode

Selects one of the following methods to control how many vias are dropped on the trunk:

  • One via per twig: Creates one via for each twig segment.
  • One via for trunk: Creates a single via connecting multiple twig segments at the same time. This is useful to avoid spacing violations when the twig segments are close and creating one via per twig generates spacing issues between vias.
  • One via if violation: Creates either one via per twig or one via for the trunk depending on the spacing violations between the vias.

Via Direction

Selects one of the following methods for the via alignment between the trunk and the twig:

  • Twig: The via enclosure is in the twig segment direction.
  • Trunk: The via enclosure is in the trunk segment direction.

Allow violations on target

Lets you generate twigs even if there are spacing violations with existing routing. By default, this option is not selected. This means that twigs are not generated, which introduces DRC errors. When the option is selected, the twigs are generated even if they create minSpacing violations and a marker is generated to explain the reason of the violation. You can then fix the violation.

Match pin width

Controls the width of the automatically created twig segments. When set to t, the width of the twig segment always matches the pin width.

Environment variable: weAutoTwigTrunkViaMode

Force all weak pins as must

Lets you establish connections with all weak pins in the design.

Targets Detection Accuracy

Relaxes the criteria, such as wider distance, components master, and so on, to help find a suitable target.

DI

This section lets you specify the naming method for the design intent dataset.

Dataset Name

Specifies the name of the dataset.

Environment variable: weSdrDIDatasetName

Dataset Naming Method

Selects one of the following methods for the naming of design intent dataset:

  • Use Dataset name: Uses the specified dataset name when the Import DI as Dataset option is selected.
  • Append date & time to Dataset Name: Appends the current date and time to the specified name of the dataset. The default name that is generated is in the format of DI_<date>_<hour>. For example, DI_
  • User defined Dataset Name: Lets you specify a new dataset name.

Environment variable: weSdrDatasetNamingMethod

EM

EAD Options

Displays the EAD Options form.

Consider scalar dynamic currents

Controls whether the peak and RMS current scalar values are visible in the EAD browser. These peak and RMS current scalar values are used by SDR for the current estimation and for the conversion of Design Intent into a dataset.

Environment variable: supportSDRDynamicDataset
