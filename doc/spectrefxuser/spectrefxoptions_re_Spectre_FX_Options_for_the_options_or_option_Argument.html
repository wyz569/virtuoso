Spectre FX Options for the options or .option Argument

The Spectre FX options can be categorized as follows:

  • High-Level Accuracy/Performance Tuning Options
  • Solver Related Options
  • RC Tuning Options
  • Device Model Options
  • Parsing Options
  • Probing and Measurement Options
  • Backannotation Flow Options
  • Miscellaneous Options
Options names that begin with sfx_ are proprietary to Spectre FX. The other options can be shared with other Spectre simulators.

High-Level Accuracy/Performance Tuning Options

The following table lists the accuracy/performance tuning options:

Option Description

sfx_optimize

Defines the parasitic optimization, including RC Reduction. Possible values are ax, mx, lx, and vx. The option can be set locally

sfx_preset

High-level preset for accuracy/performance tradeoff. Possible values are baseax, basemx, baselx, basevx, ax, mx, lx, and vx. This option can be set locally.

sfx_speed

Defines the simulation tolerance for the voltage and current calculation. The option can be set locally.

Solver Related Options

The following table lists the options related to resistors and capacitors:

Option Description

sfx_integ_method

Defines the integration method for transient simulation. Possible values are gear, gear2, traponly and euler.

sfx_speed_signal

Defines the simulation tolerance only for the signal nodes that are detected automatically by Spectre FX.

sfx_speed_power

Defines the simulation tolerance only for the power nets that are detected automatically by Spectre FX.

sfx_speed_window

Sets the option sfx_speed based on the time window during the transient simulation.

sfx_maxstep_window

Sets the transient simulation control parameter maxstep based on the time window.

sfx_time_step_errtol_factor

Specifies the factor to tune the tolerance of time step.

sfx_unit_time

Specifies the transient simulation time unit.

RC Tuning Options

The following table lists the options related to resistors and capacitors:

Option Description

sfx_ccap_cut

Defines the threshold value for the capacitors that need not be cut during partitioning.

sfx_ccap_esv

Defines the voltage threshold of the event related to the capacitors.

sfx_optimize_power

Defines the parasitic optimization only for the power nets that are detected automatically by Spectre FX.

sfx_optimize_signal

Defines the parasitic optimization only for the signal nodes that are detected automatically by Spectre FX.

sfx_rcr

Defines RC Reduction, which is controlled by sfx_optimize if no definition is given.

sfx_rmin

Resistors with value less than the specified value are shorted.

sfx_rmax

Resistors with value more than the specified value are open.

Device Model Options

The following table lists the options related to device models:

Option Description

cmi_mosfet

Lists the device names that are treated as mosfet.

sfx_support_nqs

Supports trnqsmod of bsim4 and bsim3v3, if it is defined as 1.

sfx_fast_mosfet_cap

Specifies whether to enable the fast calculation for MOSFET capacitances.

soft_bin

If set to singlemodels, it is used only on non-binned models. Possible values are singlemodels, no, and allmodels.

Parsing Options

Spectre FX supports both SPICE syntax and Spectre syntax for most of the options and the related parsing options. The following table lists only a few selected options.

Option Description

duplicate_measure

Specifies whether measures are allowed.

duplicate_module

Specifies whether duplicate module definitions are allowed.

duplicate_subckt

Specifies whether duplicate subcircuit definitions are allowed.

duplicateinstance

Specifies whether duplicate instance definitions are allowed.

duplicatemodel

Specifies whether duplicate model definitions are allowed.

duplicateports

Specifies whether duplicate ports are allowed in the subcircuit definition.

redefinedparams

Specifies whether parameters can be redefined in the netlist.

skip

Defines the simulation tolerance for the voltage and current calculation. This option can be set locally.
For detailed information, refer to Spectre Classic Simulator, Spectre APS, Spectre X, and Spectre XPS User Guide.

Probing and Measurement Options

The following table lists the options to control output statements and output format.

Option Description

dotprobefmt

Prints the .probe signal with the original name or the hierarchical name. Possible values are flat and hier.

mt_format

Specifies the format of the measurement file.

mt_separate_sweep_files

Specifies whether to separate or combine the measurement files for sweep analyses.

output_case

Outputs all signal names in lower case or upper case. Possible values are original, lower, and upper. The default value is original.

probe_global_term

Probes the implicit global terminal of the subcircuit which does not exist in the netlist but is drilled because the global node is referred to in the subcircuit. Possible values are no and yes. The default value is no.

probe_level

Specifies how to count hierarchical depths for probe statement.

sfx_compress_waveform

High-level option for waveform compression.

sfx_compress_waveform_iabstol

Detailed options for waveform compression. Separate options are provided to further tune the absolute and relative error tolerance for voltage and current signals.

sfx_compress_waveform_ireltol

sfx_compress_waveform_vabstol

sfx_compress_waveform_vreltol

sfx_lprobe_format

Specifies the location where you want to save the logic waveforms.

sfx_separate_current_waveform

Specifies whether to save a separate waveform file for current signals. By default, Spectre FX saves both voltage and current signals to the same waveform file.

sfx_waveform_flush_percentage

Defines the percentage of transient simulation to dump the waveforms to the file.

sfx_waveform_flush_minute

Defines the specified period of transient simulation to dump the waveforms to file.

Backannotation Flow Options

The control options for the parasitic backannotation flow are discussed in the section Control Options for the Parasitic Backannotation Flow.

Miscellaneous Options

Option Description

print_section

Prints the information about the library files read during the simulation.

sfx_cshunt

Specifies the value of grounded capacitors to be inserted for each nodes.

sfx_cut_floating_gate

Adjusts the partition algorithm to perform the cut for the floating gate of the MOSFET.

sfx_dump_floating_gate_node

Specifies whether to dump all the detected floating gate nodes.

sfx_floating_gate_gshunt

Specifies the value of gshunt placed on floating gate nodes.

sfx_floating_node_gshunt

Specifies the value of gshunt placed on floating nodes without conducting path to both VDD and Ground.

sfx_oscillator_node

Specifies one node of an oscillator loop. The simulator automatically starts the oscillation when the oscillator is enabled.

sfx_progress_percentage

Specifies the frequency to update the transient status based on the percentage value.
