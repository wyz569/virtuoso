Constraints Supported by I/O PAD Placer

The IO PAD Placer supports the following constraints:

  • Fixed/Locked Constraint: IO PAD Placer does not place PADs that have the fixed or locked constraints. The following table lists the placement statuses of I/Os before and after I/O placement.
    Placement status before I/O Placement Placement status after I/O Placement

    Placed

    Placed

    Firm

    Firm

    Locked

    Locked
  • Alignment Constraint: Specifies the side (TOP, BOTTOM, LEFT, RIGHT) for pad instances.
    IO PAD Placer preserves the order in which the pad instances are placed in the design. In some cases, you may need to set the PreserveOrder parameter of the alignment constraint to TRUE to get the required results.
  • Orientation Constraint: Specifies the orientation for pad instances.
  • Spacing constraints: The spacing constraint is defined in the IO PAD Placement form (Even, Abut, and Custom).
  • Placement Blockage: I/O PAD Placer respects placement blockages and pre-placed instances and does not allow you to place an instance in the design. This means while placing a PAD, if an instance is found on a location where the PAD is to be placed, the placement of the PAD is accordingly adjusted leaving the instance as is.
