Registering an Area Estimation Function to Adjust a Soft Block PR Boundary

You can adjust the PR boundary of a soft block. In the CIW, type the command given below to define an area estimator function for generating a soft block of custom block size during a Generate All From Source or an Update Components And Nets run.

Procedure(mySBAreaEstimationFunction(cellviewId)
(
list(area) | list('util f_util) | list([area] 'util f_util) | list('aspect f_aspect)
| list([area] 'aspect f_aspect) | list([area] 'width f_width) | list([area] 'height f_height)
| list('width f_width 'height f_height) | list([area] 'util f_util 'aspect 'f_aspect) 
| list([area] 'aspect f_aspect 'util f_util) | list('height f_height 'width 'f_width) 
| list([area] 'util f_util 'width f_width) | list([area] 'width f_width 'util f_util)
)

Term Description

f_area

Specifies the area of a soft block.

list(f_area)

Specifies the area of a soft block as a list.

list(f_area 'util f_util)

Specifies the area utilization of a virtual hierarchy.

The default value is specified on the form. It can be an integer or double - 25 or 25.0.

list(f_area 'aspect f_aspect)

Specifies the aspect ratio of the soft block.

The default value is specified on the form, 1.0 can be an integer or double - 1 or 1.0.

list(f_area 'util f_util 'aspect f_aspect)

Specifies both area utilization and aspect ratio of a soft block.

list(f_area 'width f_width)

Specifies the width of a soft block.

It must be a float value.

list(f_area 'height f_height)

Specifies the height of a soft block.

It must be a float value.

list('width f_width 'height f_height)

Specifies the width and height of the soft block.

list('height f_height 'width f_width)

Specifies the width and height of a soft block.
If you specify the custom area function using an incorrect syntax but specify the correct original area value, the original area value will be used.
To adjust the PR boundary of the generated soft block:
  • In the CIW, type the command given below to register an area estimator function to calculate area involving soft blocks:
    procedure(mySBAreaEstimationFunction
    (cellviewId @optional (contactArea  1.0)
  prog(((area 0.0))
    // Function body to calculate area involving soft blocks
insts = cellviewId~>instances
           foreach(ins cellview~>insts
               area = area + ins~>w * ins~>l * contactArea
           ) 
           return(area)
        ))
    leRegAreaEstimator('mySBAreaEstimationFunction"simpleAreaEst"'('cellview)' ((contactArea 1.0))"direct")
    Here, direct means that the function is applied when you choose the PR Boundary based or Bbox based area estimators on any of these forms: Adjust Boundary, Generate Layout, or Update Components And Nets.
    Alternatively, direct means the function is applied when you set the initAreaFuncName environment variable. When available, the schematic cellview is passed to the function. Else, the symbol cellview is used.
