Create Pin From Net Shapes Form

Use the Create Pin From Net Shapes form to create pins from net shapes in the layout cellview. Only top-level pins can be created using this method.

Field Description

Net in

Specifies the scope for searching net shapes in the current cellview. The available options are:

  • Cellview: Considers all the net shapes in the current cellview.
  • Selected: Considers pins on all the net shapes that you have selected.

Environment variable: modeOption

Boundary Pin

Creates pins on all net shapes that are abutted to the PR boundary.

Buried Pin

Creates pins on all net shapes that are not abutted to the PR boundary.

Consider Net Shape on Routing Layers Only

Restricts the search to the net shapes that are located on routing layers.

Create Pin on Top Layer Only

Creates pins only on the top layer shape. This option is useful when multiple shapes of the same net have different layers, and you want to create pins only on the top layer shapes. For example, if there are two net shapes A on metal layers M2 and M4, then pins are created only on the highest metal layer M4.

Create Pin on Connected Shapes

Restricts the search to those net shapes that are connected to top-level nets.

Consider Level-1 Connected Shapes

Includes net shapes that are connected to level-1 nets in the search. This option is available only when the design has level-1 shapes.

Check for Duplicate Pins

Checks for existing pins with the same names to ensure that duplicates are not created.

Schematic Aware

Ensures that pins are created only for those nets that have corresponding terminals in the schematic view. Pins are not created for internal nets.

Create Label

Creates labels over pins.

Options

Displays the Set Pin Label Text Style form, which can be used to define pin label settings.

Pin Layer

Specifies the LPP on which the pins must be created. The available options are:

  • Same as Shape LPP: Creates pins on the layers on which the net shapes are located.
  • Same Layer, Custom Purpose: Uses the same layer as the net shape, but lets you specify the purpose.
  • Custom LPP: Specifies the layer and purpose on which pins must be created.

Pin Connectivity

Sets the routing preferences to one of the following:

  • Strongly Connected (default): Specifies that the router must connect the pins internally, within the device. Consider pins Term:Pin:Fig and Term:Pin:Fig1. Here, only the pin fig name is different. The term name and pin name are the same. Such pins are called strongly connected pins.
  • Must Connect: Applies the LEF58_MUSTJOINALLPORTS property to all auto-created pin terminals, which implies that the router must connect to all pins of the selected terminal.

Preview
  • Show: Displays a visual depiction of how the pins would be created; actual pins are not created yet.
  • Hide: Hides the preview.
