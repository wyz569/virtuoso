ASCII MPP Template File Loading Returns a CIW Warning

When loading an ASCII template file, the following warning message is displayed in the CIW: 

MPP template name is replacing an existing MPP template

When you load an ASCII multipart path (MPP) template file, the system checks the names of the MPP templates in the ASCII file against the names of MPP templates that already exist in the technology file in the virtual memory. If there is a name conflict, the system displays a warning in the CIW stating that an MPP template in the ASCII file is replacing an existing MPP template.

The existing template is overwritten in virtual memory. If you do not want to overwrite the original MPP template in your binary technology library on the disk, then do not save changes to the technology file. You can also avoid overwriting existing templates by changing the duplicate names in your ASCII file.

If you want to change the names of the MPP templates in your ASCII file right away, or you want to be able to save other changes to your technology library on disk without overwriting the original template, do the following:

  1. Exit the layout software without saving technology file changes.
  2. Edit your ASCII file to change template names.
  3. Restart the layout software.
  4. Reload your ASCII MPP template file.
