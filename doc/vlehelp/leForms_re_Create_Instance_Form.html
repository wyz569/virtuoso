Create Instance Form

Use the Create Instance form to create an instance in the specified window.

Field Description

Browse

Lets you select the library, cell, and view names in the library browser.

Library, Cell, and View

Sets the library, cell, and view names of the master cell you want to place as an instance in this cellview. A thumbnail of the library, cell, and view that you have selected will be displayed on the Create Instance form. The Library, Cell, and View text fields, which can be auto-completed. When you type the characters, the drop-down list shows the names containing the matching characters at any position in the name (beginning, middle, or end of the name). Select from the drop-down list and specify the name in the text field. The drop-down list is retrieved from cds.lib. The fields are highlighted if there are no matching library or cell or view names for the characters mentioned in the field.

Names

Sets the name assigned to this instance. You can type any name unique to the cellview here or let the layout editor automatically assign instance names that begin with the letter I, followed by a number. You can enter multiple names (separated by a space) to place several instances of the same cell. If the design has EMH, you can specify a hierarchical path name for creating the instance at the desired location in the EMH.

Halo

Turns on or off the ability to define halo blockages for the instance.

Define Halo

Opens the Halo Editor Form.

Physical Only

Creates an instance in such a way such that it exists only in the block domain and not in the module domain. This option is enabled only for instances and is disabled for mosaics. Once you create an instance as a physical only instance, you cannot change its domain visibility. This attribute is used for filler (pad and core) and corner cells.

Environment variable: instPhysicalOnly

Mosaic
  • Create as Mosaic: Lets you create an instance array when you select the check box. When this option is selected, the Physical Only option is disabled.
    Environment variable: instMosaicCreation
  • Rows, Columns: Sets the number of rows and columns in an array of instances. These fields are enabled when the mosaic check box is selected.
  • Delta Y, X: Sets the spacing between rows and columns in an array of instances. The value of Delta X and Delta Y fields is calculated based on the value (BBox, P&R Boundary, or Snap Boundary) specified in the Instance Editing Mode field of the Layout Editor Options form. These fields are enabled when the mosaic check box is selected.

Parameters

Lists the instances in the Parameters section, if they have associated parameters. The orientation buttons at the bottom of the form let you change the orientation of the selected instance.

Cluster

Adds the instance being created to an existing cluster in the design.

Cluster drop-down list specifies the name of the cluster to which the instance is added. Choose an existing cluster from the drop-down list or type the name of a new cluster into the field.

Type

Specifies the type of cluster as inclusive, exclusive, or suggested.

Rotate and Flip icons
  • Rotate Left and Rotate Right buttons: Rotate an instance or array 90 degrees counterclockwise and clockwise, respectively. Alternatively, you can use the middle mouse button to perform rotation. Rotation of a mosaic is applied to the complete array.
    (Virtuoso RF Option) For SiP object types, the Rotate button is disabled. In addition, rotation using the middle mouse button during Create Instance is disabled for SiP object types. You can rotate SiP objects by changing the value of the angle parameter in the Parameters section of the Create Instance form.
  • Flip Vertical and Flip Horizontal buttons: Mirror an instance vertically or horizontally. As you place the instance, you can alternatively press Ctrl and click the middle mouse button to mirror the instance sideways/horizontally. Press Shift and click the middle mouse button to mirror the instance upside down/vertically.
    At the first click, the instance mirrors along the X axis.
    At the second click, the instance mirrors along the Y axis.
