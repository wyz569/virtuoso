Refreshing Cellviews to Reflect Updated Masters

Another user might edit a master cellview that is instantiated in the cellview that you have currently open. You can update the memory of your computer to reflect the results of such edits by using the Refresh command.

To refresh a cellview to view other users’ edits:

  1. In the layout window, choose Edit – Hierarchy – Refresh.
    • If no other users have made edits that affect the cellview, a message displays that no cellviews were refreshed.
    • If the master cells for any instances in the cellview have changed, a message prompts whether you want to update the instances.
  2. In the Refresh Cellviews form, select the instances that you want to update. Then, click OK.
    The cellview is redrawn to show the changes made to the master cells.
