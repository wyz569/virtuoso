Virtuoso Space-based Router Constraint Reference
Product Version IC23.1, June 2023

Obsolete Mixed-Signal Routing Tcl Constraints

The following Tcl constraints associated with Virtuoso® Space-based Router are no longer supported in the current release and are ignored if called.     

inputTaperRule

Specifies the name of the route spec to use for tapering to input pins.

The inputTaperRule constraint is obsolete. The current method specifies tapers using constraint groups attached to the objects:

  • inputtaper and outputtaper constraint group for bit net objects
  • taper constraint group for instance terminals, pins, and bit term objects

msMinNumCut

This constraint is obsolete. Use minNumCut instead.

msMinSpacing

This constraint is obsolete. Use minSpacing instead.

msMinWidth

This constraint is obsolete. Use minWidth instead.

msOverhang

This constraint is obsolete. Use tandemWidth instead.

outputTaperRule

Specifies the name of the route spec to use for tapering to output pins.

This constraint is obsolete. The current method specifies tapers using constraint groups attached to the objects:

  • inputtaper and outputtaper constraint group for bit net objects
  • taper constraint group for instance terminals, pins, and bit term objects.

shieldRule

Specifies the constraint group from which to get shield parameters (minSpacing and minWidth).

This constraint is obsolete. The current method for specifying shields is to create a shield constraint group for the nets that you want to shield. For more information, see Configuring Shield Wires.

shieldWidth

Specifies the width to be applied to parallel shield routes on a layer.

This constraint is obsolete. The current method uses minWidth in the shield constraint group for a net to specify the width of its parallel shields. For more information, see Configuring Shield Wires.

taperRule

Specifies the name of the constraint group that contains taper settings.

This constraint is obsolete. The current method specifies tapers using constraint groups attached to the objects:

  • inputtaper and outputtaper constraint group for bit net objects
  • taper constraint group for instance terminals, pins, and bit term objects.
