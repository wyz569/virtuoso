Extract Layout Form

Use the Extract Layout form to run connectivity extraction on the layout design.

Field Description

Options

Opens the Connectivity tab of the Connectivity form, where you can set the extraction parameters for the design.

Scope

This section lets you choose how much of the design is extracted.

Current Cellview

Extracts only the top level of the current design. Layout XL also supports extraction of read-only cellviews.

Current Cellview and Cellviews in Hierarchy

Extracts the selected cellviews – including real-only cellviews – in the hierarchy along with the current top-level cellview. The extraction is performed bottom-up, starting with the cellview lowest in the hierarchy and ending with the top-level cellview. You can select the cellviews to extract in the Cellviews in Hierarchy pane.

Area

Specifies the area of the design to be extracted. Specify the coordinates of the bounding box of the area to be extracted in the Bounding Box field, either by typing the coordinates directly into the field or by clicking Select Area and drawing the bounding box on the canvas.

Unverified Areas

Extracts connectivity only from areas that were not interactively verified by the connectivity extractor.

Nets

Extracts connectivity from the nets specified in the adjacent text field. The net names typed in the text field must be space-separated.

Selected Nets

Extracts connectivity from all the selected nets. When Selected Nets is selected, any net names specified for extraction are ignored. Only those nets that are selected are extracted, irrespective of their being specified for extraction in the Nets field.

Report Type

This section lets you choose the scope of the extraction report to display.

None

Displays no report.

Simple

Displays a basic extraction report indicating the number of opens, shorts, and violations.

Full

Displays a detailed extraction report covering details about the extracted cellview, extracted nets, types of annotations displayed and the time taken to extract each value. In addition, the report displays the basic information displayed in a simple report.

Cellviews in Hierarchy

This section lets you specify which hierarchical cellviews are extracted. The controls in the Cellviews in Hierarchy group box are enabled only when the extraction scope is set to Current Cellview and Cellviews in Hierarchy.

From Selected Instances

Displays only those hierarchical cellviews that belong to the selected instances from the current, edited cellview.

Edited Only

Displays only those hierarchical cellviews that belong to the cellviews that have been edited since the last extraction run.

Search Depth

Specifies how far down the hierarchy to look for the cells to extract.

Select By

Lets you filter the entries in the list using the library, cell, or view names. Type in the first characters of the names you want to select. The filter mechanism has no wildcard capability. To select a cell named ‘resistor’, you must type ‘r’, ‘re’, ‘res’, and so on. If you type ‘sis’, the cell is not selected.

You can click Select All to select all the cellviews listed or click Deselect All to deselect all the cellviews listed.

Save Extracted Cellviews

Saves the extracted cellviews automatically.
