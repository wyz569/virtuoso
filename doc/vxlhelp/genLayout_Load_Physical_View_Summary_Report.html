Load Physical View Summary Report

After running the Load Physical View command, a summary report is displayed in the CIW, which provides details about the information copied from the source cellview to the target cellview.

.

This report includes the following information:

  • Names of:
    • Source cellview
    • Target cellview
  • Names of instances that were:
    • Updated in the target cellview.
    • Missing in the target cellview, but present in the source cellview.
    • Present in the target cellview, but missing in the source cellview.
  • Names of physical-only instances that were:
    • Updated
    • Added
    • Present in the target cellview, but not in the source cellview
  • Names of re-mastered Pcell instances
  • Names of pins that were:
    • Updated
    • Created
  • Whether the following attributes were created or updated:
    • PR boundary
    • Snap boundary
  • Number of the following attributes created or updated:
    • Halos
    • Blockages
    • Rows
    • Row regions
    • Placement areas
    • Constraints
  • Number of the following attributes created:
    • Shapes
    • Wires
    • FigGroups
    • Routes
    • Count
    • Site patterns
    • Track patterns
  • Whether the following attributes were updated:
    • Cell type
    • Block type
    • Symmetry
    • Width Spacing snap patterns

If the number of instances to be updated in the target cellview exceeds 100, an abridged report can be displayed.
