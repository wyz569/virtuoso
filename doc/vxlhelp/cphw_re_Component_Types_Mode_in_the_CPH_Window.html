Component Types Mode in the CPH Window

Component Types Mode lets you view the library-level component types. In addition, this mode lets you create, edit, and remove the design-level component types that identify the NMOS and PMOS transistor cells in the design and set the parameters for device chaining and folding. The table also has a shortcut menu that lets you easily access the required commands; see Shortcut Menu for the Component Types Table

Field

Description

Cells

Shows the cells in the design organized by component type. If a cell is not assigned to a component type, it is put in the No component type folder.

Physical Library

Shows the name of the library containing the physical representation of the logical leaf cell.

Physical Cell

Shows the physical equivalent of the logical leaf cell.

Attributes

Lets you edit the attributes for the selected component type.

These controls are enabled only when you have a single component type selected in the Cells table

Component class

Specifies the type of devices in the component type. Valid values are:

Leave the field blank to set component class as undefined.

  • Choose PMOS or NMOS if you are using the MOS component-assisted row generation. The Views field is disabled in this case.
  • Choose PFIN or NFIN if you are using FinFET devices.
  • Choose RESISTOR, CAPACITOR, or INDUCTOR to recognize the split parameter on these devices when manually folding them.
    The information is used by the Layout XL Check and Update commands to verify if the parameter has been split and if the split has been accurate.
  • Choose STDCELL if you are using the Standard Cell component-assisted row generation. All other settings are disabled in this case.

  • Choose STDSUBCONT or FILLER for substrate contacts and filler cells respectively. Only the Views field is enabled; everything else is disabled in this case.
  • Choose BOUNDARYCELL for boundary cells. Only the Views and Edge type fields are enabled; everything else is disabled in this case.
  • Choose ROUTING for routing only cells, which are cells that contain no devices
  • Choose GUARDRING for guard-ring cells, which are cells that prevent unintended signal leakage to or from the components they surround.
  • (Virtuoso Advanced Node Option for GAA) Choose NGAA or PGAA if you are using Gate All Around transistor devices. These component classes are available in Layout EXL or higher tiers.

Active layer

Specifies the diffusion layer-purpose pair for NMOS and PMOS devices. The drop-down lists all the layers from the technology file for which the function is set to one of the following: pdiff, ndiff, pwell, nwell, pplus, nplus, or diffusion.

This parameter is required for NMOS and PMOS component types to control device chaining and folding during placement. The Custom Digital Placer also uses this layer to determine the spacing rules to be used when seeding the defined spacing type in the fields on the Placement Planning (Assisted CMOS) form. The Calculate Estimates command for Assisted CMOS placement uses the spacing rule between adjacent MOS device chains to achieve more accurate estimates.

Width parameter

Specifies the name of the transistor width parameter on the device master cell. The default name is w. If you use a different name for the transistor width parameter, type that name into the text field. You must set this parameter for NMOS and PMOS component types even if the devices in the component type will not be folded.

Length parameter

Specifies the name of the transistor length parameter on the device master cell. The default name is l. If you use a different name for the transistor width parameter, type that name here.

Split parameter

Specifies the name of the parameter to be split.

Folding minimum

Specifies the minimum width that can be used when folding transistors with width greater than the folding threshold.

Folding threshold

Specifies the maximum width beyond which transistors are automatically folded. You must set this parameter for NMOS and PMOS component types even if the devices in the component type will not be folded. The value applies globally to all devices within the scope of the component type definition (that is, the library containing the current cell or just the current cell).

Folding step

Specifies the available width values that lie between the Folding minimum and the Folding threshold values and can be used for folding.

If the Folding min = 0.3u, Folding threshold = 0.9u, and the Folding step = 0.3u, the following commands will create or update devices to be 0.3u, 0.6u, or 0.9u wide.

  • Generate All From Source
  • Generate Folded Devices
  • Update Components And Nets
  • Update Layout Parameters

Edge type

Specifies the edge along which the boundary cells can be placed around core cells.It is supported only for the cells that have their Component class set to BOUNDARYCELL. Choose the edge type from: Left, Right, LeftTop, LeftBottom, RightTop, and RightBottom.

Views

Specifies the view name to be used when defining a standard cell substrate contact or filler cell. This parameter is valid only when the Component class is set to STDSUBCONT or FILLER.

Orientation parameter

Reads the orientation value, if defined in the LAM file. Else, specifies the orientation value for the selected component type to be used during generation.

The orientation value defined using the environment variable, lxGenerationOrientation, is considered when the value is not already defined in the LAM file or using the Orientation parameter field.

Components parameters

Specifies multiple component parameters for a single cell to allow the same cell to be assigned different component types. To determine the supported component types for a cell, the component parameters are compared to the CDF parameters on the device master cell.

The component parameters for a design master cell can only be specified or edited using the LAM file. The value is not editable using the Configure Physical Hierarchy window.

Terminal Names

Lets you edit the default names used to identify the Drain, Gate, Source, and Bulk terminals in the cells assigned to the component type. The names are not case-sensitive. The defaults are: drain and D; gate and G; source and S; and bulk and B, respectively.

Number of fins

Specifies the name of the number of fins parameter on the FinFET master cell. You can choose nFins, nfins, NFINS, or an empty string from the drop-down list. You can also specify a default value for this field by defining it in the LAM File Component Type Section.

The value specified in this field is used to determine the whole number of fins that a FinFET device should be folded into during layout generation using commands such as Generate All From Source.

To fold or chain FinFET devices based on number of fins, the Number of fins parameter and the Folding threshold fins values are required. Width parameter, Folding threshold, Views, and Width per fin are disabled in this case.

This attribute supports only the NFIN and PFIN component types. The parameter is required for driving FinFET chaining and folding.

Number of fins

Specifies the name of the number of fins parameter on the FinFET master cell. You can choose nFins, nfins, NFINS, or an empty string from the drop-down list. You can also specify a default value for this field by defining it in the LAM File Component Type Section.

The value specified in this field is used to determine the whole number of fins that a FinFET device should be folded into during layout generation using commands such as Generate All From Source.

To fold or chain FinFET devices based on number of fins, the Number of fins parameter and the Folding threshold fins values are required. Width parameter, Folding threshold, Views, and Width per fin are disabled in this case.

This attribute supports only the NFIN and PFIN component types. The parameter is required for driving FinFET chaining and folding.

Width per fin

Specifies the width of each fin.

To fold or chain FinFET devices based on the Width parameter, the Width per fin, and the Folding threshold fins values are required. Folding threshold, Views, and Number of fins are disabled in this case, as displayed in the figure below.

If automatic folding is driven by the Width parameter, it uses the product of the Width per fin value and the Folding threshold fins value to achieve a whole number of fins per fold.

This attribute supports only the NFIN and PFIN component types. The parameter is required for driving FinFET chaining and fold

Folding threshold fins

Specifies the maximum number of fins beyond which FinFETs are automatically folded.

This attribute supports only the NFIN and PFIN component types. The parameter is required for driving FinFET chaining and folding.

Shortcut Menu for the Component Types Table

The shortcut commands, also called the shortcut menu commands, are available on right-clicking a cell within the Components Type table. These commands are enabled and disabled dynamically depending on the cell that is selected in the table view.

Command Name

Description

Cells

 Lets you access the shortcut menu commands that you can use to perform specific tasks in the Cells table.

Add component type

Opens the Create Component Type dialog, which lets you create a new component type to be added to the table view.

See Defining a Design-Level Component Type.

Remove component type

Removes the selected component type from the table view. You can remove only design-level component types. Evicted cellviews are put into the No component type folder.

See Removing a Component Type.

Move cells

Opens the Move Cells dialog, which lets you move selected cells between component types.

See Defining a Design-Level Component Type.

Show cells

Choose In design (the default) only to show the physical cells that will be generated in the layout based on the current physical configuration view.

Choose one or more of the library names to toggle the display of cells from that library. The submenu lists all the libraries from the library definitions file, cds.lib.

See Component Types Cells Table.f
