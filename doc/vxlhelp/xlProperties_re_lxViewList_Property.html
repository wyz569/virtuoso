lxViewList Property

Description

This property is obsolete in the current release. It is automatically converted to the new Layout XL schema underlying physical hierarchy configuration.

To tell Layout XL where to descend into the design hierarchy to find cells at a lower level,

  1. From the layout window menu bar, choose Launch – Configure Physical Hierarchy.
  2. In the Instances or Cells table, type the view names you require into the Inherited View List field for the instance or cell in question.

To prevent Layout XL from automatically upreving lxViewList (and other “lx” properties such as lxStopList and lxUseCell), and to continue using the schematic property instead:

Set the CPH shell environment variable, CPH_USE_SCHEM_PROP, to 1, before launching Layout XL. If CPH_USE_SCHEM_PROP is set to 1, Layout XL continues to read and honor any updates made to the properties in the schematic.

If you have already upreved the lx properties or made edits to the physical bindings (lxUseCell), physical stop list (lxStopList), or view list (lxViewList), then the value set in CPH takes precedence. To continue using the corresponding schematic properties, you must remove the edits from CPH.
