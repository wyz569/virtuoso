Check Against Source Form

Use the Check Against Source form to check the differences between the schematic and the layout. Use Connectivity – Check – Against Source (CAS) command to invoke the form.

Field Description

Check

This section lets you specify the types of layout versus schematic checks you want to run.

Selected only

Runs the Check Against Source command only on a selected set of instances.

Environment variable: casSelected

Unbound instances

Reports the layout instances that are not bound to the schematic instances and the schematic instances that are not yet generated in the layout.

Environment variable: casDisplayUnboundInsts

Masters

Reports the instance masters that are missing or mismatched between the schematic and layout.

Environment variable: casDisplayMastersMismatch

Connectivity

Reports connectivity issues on top-level pins and global nets, mismatched or missing terminals and instance terminals, and unbound nets in the layout.

Environment variable: casDisplayConnMismatch

Parameters

Reports issues with CDF parameters and properties set on the schematic and the layout views. The parameters (and properties) to be compared are set on the Parameters tab of the Connectivity form.

If the checkParamsOnly environment variable is set to nil, the Parameters field on the form updates to Parameters and properties, reporting both the parameter and property differences.

Environment variable: casDisplayParamsMismatch

Names

Reports mismatches between the schematic and the layout instance and net names.

Environment variable: casDisplayNameMismatch

Dummy devices

Reports dummy instances that are unbound, which means they have no correspondence to the schematic devices and have the ignore property set.

These are dummies instances that are good candidates for backannotation.

Environment variable: casDisplayDummyMismatch

Bus terminals

Reports mismatches between the implicit bus terminals that exist in the layout cellview and the explicit bus terminals that exist in the schematic cellview. The related bus terminal markers generated in the CAS tab of the Annotation Browser assistant can be interactively fixed using the Update All Bus Terminals shortcut menu.

Environment variable:casDisplayBusTerminalsMismatch

Design Intent

Checks if Design Intent synchronization is needed during CAS run. By default, this option is deselected.

Design planner

Reports mismatches between the virtual hierarchy in the schematic and layout views. This option is available only when Virtuoso Layout Suite EXL or higher license is checked out.

Output

This section lets you specify how the report should be displayed.

Open workspace

Opens the CAS workspace at the end of a Check Against Source run. By default, this option is ON.

Environment variable: casOpenWorkspace

Display info window

Prints the report in a separate Info window.

By default, the schematic versus layout differences are reported in the form of markers generated in the CAS tab of the Annotation Browser assistant.

Environment variable: casInfoWindow

Overwrite log file

Clears the log file of all the information from the previous CAS run and populates it with the information from the new run. If the option is deselected, the report from a new CAS run is appended to the specified log file.

Environment variable: casAppendToLogFile

Log File

Specifies the name of the file to which all the Check Against Source messages are written. You can use the browse button to locate an existing file.

Environment variable: casLogFileName

Glyphs

This section lets you specify the display associated with each generated glyph.

Display

Controls whether or not the markers generated during a Check Against Source run are visually indicated on the layout canvas using an appropriate glyph.

Environment variable: casGlyphs

Info balloon

Controls the display of the information balloon associated with each generated glyph. Choose between displaying the information balloon on click or on hover.
