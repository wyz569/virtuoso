Coloring Synchronization in Synchronous Clones

Layout XL supports coloring synchronization for synchronous clones. This means that if the mask color on one of the synchronous clones is manually updated, corresponding shapes in the other synchronous clones are automatically updated and synchronized. For example, if a mask1Color color-locked shape in a synchronous clone has its mask color updated to mask2Color, all the corresponding mask1Color color-locked shapes in other synchronous clones also change their mask color to mask2Color.

To prevent the color state and color updates on color-locked shapes from being carried to the corresponding shapes in other synchronous clones, set the enableCloneColoringSupport environment variable to nil.

Consider some design situations that illustrate the extent of color state or color synchronization possible, depending on the manual edit made to a shape in a synchronous clone:

  • Scenario 1 — The color state of a shape in the synchronous clone is switched from unlock to lock
    Because Layout XL supports fully synchronizing those shapes that are locked on a color, in this case, both the color state and color are synchronized on equivalent shapes in other synchronous clone(s).
  • Scenario 2 — The color state of a shape in the synchronous clone is switched from lock to unlock
    In this case, the color state is synchronized on equivalent shapes in other synchronous clones but the color is not synchronized by default if cloneSynchronizeUnlockedColor is set to the default = "no". For the color to be synchronized, the environment variable must be set to "shape only" or "cut and shape".
  • Scenario 3 — The color of a “color locked” shape is changed
    In this case, the color is synchronized on equivalent shapes in other synchronous clones. No change is made to the color state.
  • Scenario 4 — The color of a “color unlocked” shape is changed
    No synchronization takes place by default if cloneSynchronizeUnlockedColor is set to "no". For the color to be synchronized, the environment variable must be set to "shape only", "cut and shape", or "cut only".

To change the color state of a shape in a synchronous clone from lock to unlock or vice versa, you must Edit in Placethe synchronous clone. Such updates to shapes in a synchronous clone are not supported when operating from the top cellview. Conversely, if the shape in question is not a member of a synchronous clone family, color and other updates are possible from the top cellview just as for other regular shapes.

By design, Layout XL keeps the clone families synchronized by preventing any coloring updates from the top cellview that can impact the shapes inside synchronous clones.

By default, only locked colors are synchronized, but in a lockless flow, you can synchronize unlocked colors by setting the value of cloneSynchronizeUnlockedColor environment variable to "cut only", or "shape only", or "cut and shape".

When you do this, all locked and unlocked changes are synchronized with other clones during Edit in Place. For example, if cloneSynchronizeUnlockedColor is set to "cut and shape" and you change an unlocked cut color to mask3 in one clone, the cut from the other via in the other clone will also be updated to mask3.

The pre-existing unlocked color differences are still skipped if the clone checker detects a difference in pre-existing clones about unlocked color, the clones are not desynchronized.
