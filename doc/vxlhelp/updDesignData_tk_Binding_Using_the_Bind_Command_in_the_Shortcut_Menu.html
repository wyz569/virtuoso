Binding Using the Bind Command in the Shortcut Menu

To manually bind an instance, mosaic, or terminal:

  1. In the layout navigator or canvas, select the layout instance to be bound.
    You can also use this procedure to bind selected mosaics and terminals.
  2. In the schematic navigator assistant or canvas, select the instance to which the selected layout instance needs to be bound.
  3. In the layout canvas, right-click the selected instance and choose Bind from the shortcut menu.
  4. The selected schematic and layout instances are bound and the XL Status column of the layout navigator assistant displays the binding status, as appropriate.
    • If the environment variable bindUpdateName is set to t, the name of the layout instance is automatically updated after binding to match the name of the corresponding schematic instance.
    • After binding, the logical (or schematic instance) connectivity is automatically propagated to the layout instance. But, if there is a terminal mismatch between the schematic and layout instances, the Assign layout instance terminals form is displayed so that layout nets can be selected and assigned manually to instance terminals, as appropriate.
    • If the manual binding results in complex binding between the selected schematic and layout instances, a warning message is displayed in the CIW to indicate the same.

To unbind a layout instance, right-click the instance in the layout navigator or canvas and choose Unbind from the shortcut menu.
