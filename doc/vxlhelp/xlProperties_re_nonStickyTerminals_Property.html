nonStickyTerminals Property

nonStickyTerminals string l_terminals

Description

The nonStickyTerminals property is used for an individual instance or instance master. When the connectivity is removed from the instance terminals, the connectivity extractor does not report any open or short markers.

Layout instance Add Property form

Name

nonStickyTerminals

Type

string

Value

l_terminals

Examples

In the following example:

  • Instance I0 has three terminals S, D and G
  • nonStickyTerminals is set on terminal S and D of instance I0

Here, S_1 and S, G and G_1 are the must connect terminals.

Consider that netA is set on three terminals using the Connectivity – Nets – Propagate command.

Interactive or the batch connectivity extractor removes the connectivity on S, S_1, and D. On terminal G, the connectivity is not removed.

In the following example with nonStickyTerminals S and D, two shapes are created on netA. After interactive or batch extraction, S, S_1, D, G, and G_1 are assigned on netA. If the shapes are deleted from netA, then after interactive or batch extraction, the net is removed from S, S_1, and D but not from G.
