LAM File Component Type Section

<COMPONENT_TYPE_GROUPS>
  <LIB NAME="libName" >
    <COMPONENT_TYPE_GROUP NAME="compTypeName" >
      <COMP_CLASS>PMOS | NMOS | NFIN | PFIN | STDCELL | STDSUBCONT | BOUNDARYCELL | FILLER</COMP_CLASS>
      <WIDTH>WIDTH</WIDTH>
      <LENGTH>LENGTH</LENGTH>
      <SPLIT>SPLIT</SPLIT>
      <ORIENTATION>device_orientation</ORIENTATION>
      <DRAIN>list_of_drainTermNames</DRAIN>
      <GATE>list_of_gateTermNames</GATE>
      <SOURCE>list_of_sourceTermNames</SOURCE>
      <BULK>list_of_bulkTermNames</BULK>
      <ACTIVE_LP>layerPurposePairName</ACTIVE_LP>
      <FOLD_MIN>float</FOLD_MIN>
      <FOLD_THRESH>float</FOLD_THRESH>
      <COMPTYPE_PARAMS>list_of_parameters_and_value_pairs</COMPTYPE_PARAMS>
      <FOLD_STEP>float</FOLD_STEP>
      <VIEWS>list_of_viewNames</VIEWS>
      <COMPTYPE_PARAMS>list_of_parameters</COMPTYPE_PARAMS>
      <NUM_FINS>number_of_fins_ParameterName</NUM_FINS>
      <FOLD_THRESH_FINS>int</FOLD_THRESH_FINS>
      <WIDTH_FIN>float</WIDTH_FIN>
    </COMPONENT_TYPE_GROUP>
  </LIB>
</COMPONENT_TYPE_GROUPS>

The following attributes in the LAM file above are supported for FinFET devices:

    • <COMP_CLASS>NFIN | PFIN</COMP_CLASS>
    • <NUM_FINS>number_of_fins_ParameterName</NUM_FINS>
    • <FOLD_THRESH_FINS>int</FOLD_THRESH_FINS>
    • <WIDTH_FIN>float</WIDTH_FIN>

Description

Specifies the library-level component types, which identify NMOS and PMOS transistor cells and sets the parameters for device chaining and folding.

For FinFET devices, the Component Type section of the LAM file also specifies the library-level component types that identify NFIN and PFIN FinFET devices and sets the parameters for chaining and folding for these FinFET devices.

The LAM file Component Type section also controls how components are assigned to rows when using the Virtuoso Custom Digital placer. The LAM file component type settings are inherited by all the cells in the specified library. They can be overridden, but not overwritten in the Configure Physical Hierarchy window.

By default, CPH only displays used cells for a design. If you want to display in CPH the cell assignments that are made in LAM (although the cells are not being used), do this:

  • In CPH Cells tab, right-click the library for which you made the cell assignments in LAM and choose Show Cells.
    CPH displays the LAM cell assignments for the library.

The component type settings defined in the LAM file are created automatically during the logical library conversion. This takes the component types stored in CDF files or (if none exist in CDF) in the data.dm file in the library directory and converts them to LAM file format.

If the same component type is defined with identical attributes in two different libraries, the definitions are merged into a single definition in the LAM file. All required cells are then assigned to that single component type.

You can also add new component type groups using the syntax described in this section. for more information, see Data Conversion Flow for the Layout XL Schema.

When using multiple SKILL APIs for working with component types, it is recommended that you checkout a Layout XL license beforehand to enhance the system performance. But if you have a CPH window or an XL session open already, the XL license will already be checked out.

After you have defined the component type group, you must specify the cells that are assigned to this group using the <COMPONENT_TYPE> statement. See LAM File Mapping Section for more information.

Statements

Statement

Description

<COMPONENT_TYPE_GROUPS>

Encloses the library-level component type definitions in the LAM file.

<LIB NAME="libName" >

Specifies the name of the library to which the component type definitions apply.

<COMP_CLASS>PMOS | NMOS |PFIN | NFIN | RESISTOR | CAPACITOR | INDUCTOR | STDCELL | STDSUBCONT | BOUNDARYCELL  |  FILLER</COMP_CLASS>

Specifies the type of devices in the component type. Valid values are PMOS, NMOS, PFIN, NFIN, RESISTOR, CAPACITOR, INDUCTOR, STDCELL, STDSUBCONT, BOUNDARYCELL, FILLER. Leave blank to set to undefined.

PFIN and NFIN are valid component types only for designs using FinFET devices.

  • Choose PMOS or NMOS if you are using the placer’s MOS component-assisted row generation. The <VIEWS> statement is ignored.
  • Choose RESISTOR, CAPACITOR, or INDUCTOR to recognize the split parameter on these devices when manually folding them.
  • The information is used by the Layout XL Check and Update commands to verify if the parameter has been split and if the split has been accurate.
  • Choose PFIN or NFIN if you are using FinFET devices for generation.
  • Choose STDCELL if you are using the placer’s Standard Cell component-assisted row generation. All other settings are ignored.
  • Choose STDSUBCONT or FILLER for substrate contacts and filler cells, respectively. Only the <VIEWS> statement is considered; everything else is ignored.
  • Choose BOUNDARYCELL for boundary cells. Only the <VIEWS> statement is considered; everything else is ignored.

GUI: Component class

<WIDTH>WIDTH</WIDTH>

Specifies the name of the transistor width parameter on the device master cell. The default name is w. If you use a different name for the transistor width parameter, type that name here.

You must set this parameter for NMOS and PMOS component types even if the devices in the component type will not be folded.

GUI: Width parameter

<LENGTH>LENGTH</LENGTH>

Specifies the name of the transistor length parameter on the device master cell. The default name is l. If you use a different name for the transistor length parameter, type that name here.

GUI: Length parameter

<SPLIT>SPLIT</SPLIT>

Specifies the name of the parameter to split.

GUI: Split parameter

<ORIENTATION>device_orientation</ORIENTATION>

For the selected component class, specifies the orientation to be used during layout generation.

<DRAIN>list_of_drainTermNames</DRAIN>

Specifies the default names used to identify the Drain terminal in the cells assigned to the component type. The names are not case-sensitive. The default names are drain and D.

Separate each name with a single space. Do not enclose the list in quotation marks.

GUI: Drain

<GATE>list_of_gateTermNames</GATE>

Specifies the default names used to identify the Gate terminal in the cells assigned to the component type. The names are not case-sensitive. The default names are gate and G.

Separate each name with a single space. Do not enclose the list in quotation marks.

GUI: Gate

<SOURCE>list_of_sourceTermNames</SOURCE>

Specifies the default names used to identify the Source terminal in the cells assigned to the component type. The names are not case-sensitive. The default names are source and S.

Separate each name with a single space. Do not enclose the list in quotation marks.

GUI: Source

<BULK>list_of_bulkTermNames</BULK>

Specifies the default names used to identify the Bulk terminal in the cells assigned to the component type. The names are not case-sensitive. The default is bulk and B.

Separate each name with a single space. Do not enclose the list in quotation marks.

GUI: Bulk

<ACTIVE_LP>layerPurposePairName</ACTIVE_LP>

Specifies the diffusion layer-purpose pair for NMOS and PMOS devices. This parameter is required for component types of the component class NMOS and PMOS to control device chaining and folding during placement.

For example, to specify the ndiff drawing layer as the diffusion layer, type the following.

<ACTIVE_LP>ndiff drawing</ACTIVE_LP>

Separate the layer and purpose names with a single space. Do not enclose the names in quotation marks.

GUI: Active layer

<FOLD_MIN>float</FOLD_MIN>

Specifies the minimum width that can be used when folding transistors with width greater than the folding threshold.

GUI: Folding minimum

<FOLD_THRESH>float</FOLD_THRESH>

Specifies the maximum width beyond which MOS devices are automatically folded.

Note: You must set this parameter for NMOS and PMOS component types even if the devices in the component type will not be folded.

GUI: Folding threshold

<COMPTYPE_PARAMS>list_of_parameters_and_value_pairs</COMPTYPE_PARAMS>

Specifies the parameters names and values required for the cell to belong to this component type.

GUI: Component parameters

<FOLD_STEP>float</FOLD_STEP>

Specifies the available width values that lie between the Folding minimum and the Folding threshold values and can be used for folding.

GUI: Folding step

<VIEWS>list_of_viewNames</VIEWS>

Specifies the view name to be used when defining a standard cell substrate contact or filler cell. This parameter is valid only when the component class is either STDSUBCONT or FILLER.

Separate each name with a single space. Do not enclose the list in quotation marks.

GUI: Views

<NUM_FINS>number_of_fins_parameterName</NUM_FINS>

Specifies the name of the Number of fins parameter on the FinFET device master cell.

NUM_FINS must be set for FinFET devices that are driven for folding by number of fins.

GUI: Number of fins

<WIDTH_FIN>float</WIDTH_FIN>

Specifies the width per fin value such that the folding threshold for automatic folding is set using the product of the width per fin value and the folding threshold as a number of fins.

Set this parameter to fold FinFET devices driven by setting the Width parameter.

GUI: Width per fin

<FOLD_THRESH_FINS>int</FOLD_THRESH_FINS>

Specifies the folding threshold in terms of number of fins.

FOLD_THRESH_FINS must be set to drive automatic folding of FinFET devices.

GUI: Folding threshold fins

<COMPTYPE_PARAMS>list_of_parameters</COMPTYPE_PARAMS>

Specifies multiple component parameters for a single cell to allow the same cell to be assigned different component types. To determine the supported component types for a cell, the component parameters are compared to the CDF parameters on the device master cell.

The component parameters for a design master cell can only be specified or edited using the LAM file. The value is not editable using the Configure Physical Hierarchy window.

Let us consider some examples depicting the use of the various attributes.

Example 1 LAM file depicting the use of various component types

<COMPONENT_TYPE_GROUPS>
  <LIB NAME="lib1" >
    <COMPONENT_TYPE_GROUP NAME="nmos" >
      <COMP_CLASS>NMOS</COMP_CLASS>
      <WIDTH>w</WIDTH>
      <ORIENTATION>R90</ORIENTATION>
      <DRAIN>d</DRAIN>
      <GATE>g</GATE>
      <SOURCE>s</SOURCE>
      <BULK>b</BULK>
      <ACTIVE_LP>pdiff drawing</ACTIVE_LP>
      <FOLD_MIN>0.3</FOLD_MIN>
      <FOLD_THRESH>0.9</FOLD_THRESH>
      <FOLD_STEP>0.3</FOLD_STEP>
      <VIEWS>layout filler</VIEWS>
    </COMPONENT_TYPE_GROUP>
    <COMPONENT_TYPE_GROUP NAME="pmos" >
      <COMP_CLASS>PMOS</COMP_CLASS>
      <WIDTH>w</WIDTH>
      <DRAIN>d</DRAIN>
      <GATE>g</GATE>
      <SOURCE>s</SOURCE>
      <BULK>b</BULK>
      <ACTIVE_LP>ndiff drawing</ACTIVE_LP>
      <FOLD_THRESH>0.9</FOLD_THRESH>
      <VIEWS>layout</VIEWS>
    </COMPONENT_TYPE_GROUP>
  </LIB>
  <LIB NAME="lib2" >
    <COMPONENT_TYPE_GROUP NAME="lib2" />
  </LIB>
</COMPONENT_TYPE_GROUPS>

The example above defines three library-level component types.

  • One called nmos and one called pmos for a library called lib1.
  • One called lib2 for a library called lib2.

The Folding minimum, Folding threshold, and Folding step values set in the LAM file result in the following commands to create or update devices to be 0.3u, 0.6u, or 0.9u wide.

  • Generate All From Source
  • Generate Folded Devices
  • Update Components And Nets
  • Update Layout Parameters

Let us now consider some examples depicting the component types section of LAM files for FinFET devices.

Example 1 LAM File for FinFET devices that have the Number of fins parameter set

<PHYSICAL>
 <COMPONENT_TYPE_GROUPS>
  <LIB NAME=“cdn20”>
   <COMPONENT_TYPE_GROUP_NAME=“nfin”>
    <COMP_CLASS>NFIN</COMP_CLASS>
    <NUM_FINS>nFins</NUM_FINS>
   </COMPONENT_TYPE_GROUP>
   <COMPONENT_TYPE_GROUP_NAME=“pfin”>
    <COMP_CLASS>PFIN</COMP_CLASS>
    <NUM_FINS>pFins</NUM_FINS>
   </COMPONENT_TYPE_GROUP>
  </LIB>
 </COMPONENT_TYPE_GROUPS>
 <MAPPINGS>
  <LIB NAME=“cdn20”>
   <CELL NAME=“nfinFET”>
    <COMPONENT_TYPE>
     <NAME>nfin</NAME>
     <LIB>cdn20</LIB>
    </COMPONENT_TYPE>
   </CELL>
   <CELL NAME=“pfinFET”>
    <COMPONENT_TYPE>
     <NAME>pfin</NAME>
     <LIB>cdn20</LIB>
    </COMPONENT_TYPE>
   </CELL>
  </LIB>
 </MAPPINGS>
</PHYSICAL>

The example above defines the components of the class NFIN and PFIN that have the Number of fins parameter set as nFins and pFins, respectively.

Example 2 LAM File for FinFET devices that have the Width parameter set

<PHYSICAL>
 <COMPONENT_TYPE_GROUPS>

    <WIDTH>w</WIDTH>
    <WIDTH_FIN>0.3</WIDTH_FIN>
   </COMPONENT_TYPE_GROUP>
  </LIB>
 </COMPONENT_TYPE_GROUPS>
 <MAPPINGS>
  <LIB NAME=“cdn20”>
   <CELL NAME=“nfinFET”>
    <COMPONENT_TYPE>
     <NAME>nfin</NAME>
     <LIB>cdn20</LIB>
    </COMPONENT_TYPE>
   </CELL>
   <CELL NAME=“pfinFET”>
    <COMPONENT_TYPE>
     <NAME>pfin</NAME>
     <LIB>cdn20</LIB>
    </COMPONENT_TYPE>
   </CELL>
  </LIB>
 </MAPPINGS>
</PHYSICAL>

The example above defines the components of the class NFIN and PFIN that have the Width parameter set as w.

Example 3 LAM File for Automatic folding of FinFET devices based on Number of fins 

<PHYSICAL>
 <COMPONENT_TYPE_GROUPS>
  <LIB NAME=“cdn20”>
   <COMPONENT_TYPE_GROUP_NAME=“nfin”>
    <COMP_CLASS>NFIN</COMP_CLASS>
    <NUM_FINS>nFins</NUM_FINS>
    <FOLD_THRESH_FINS>2</FOLD_THRESH_FINS>
   </COMPONENT_TYPE_GROUP>
   <COMPONENT_TYPE_GROUP_NAME=“pfin”>
    <COMP_CLASS>PFIN</COMP_CLASS>
    <NUM_FINS>pFins</NUM_FINS>
    <FOLD_THRESH_FINS>3</FOLD_THRESH_FINS>
   </COMPONENT_TYPE_GROUP>
  </LIB>
 </COMPONENT_TYPE_GROUPS>
 <MAPPINGS>
  <LIB NAME=“cdn20”>
   <CELL NAME=“nfinFET”>
    <COMPONENT_TYPE>
     <NAME>nfin</NAME>
     <LIB>cdn20</LIB>
    </COMPONENT_TYPE>
   </CELL>
   <CELL NAME=“pfinFET”>
    <COMPONENT_TYPE>
     <NAME>pfin</NAME>
     <LIB>cdn20</LIB>
    </COMPONENT_TYPE>
   </CELL>
  </LIB>
 </MAPPINGS>
</PHYSICAL>

The example above indicates that the FinFET devices have the number of fins parameter set. As a result, the devices will automatically fold based on the fins per fold when the folding threshold defined by the FOLD_THRESH_FINS is reached.

Example 4 LAM File for Automatic folding of FinFET devices based on fin width 

<PHYSICAL>
 <COMPONENT_TYPE_GROUPS>
  <LIB NAME=“cdn20”>

   </COMPONENT_TYPE_GROUP>
   <COMPONENT_TYPE_GROUP_NAME=“pfin”>
    <COMP_CLASS>PFIN</COMP_CLASS>
    <WIDTH>w</WIDTH>
    <WIDTH_FIN>0.3</WIDTH_FIN>
    <FOLD_THRESH_FINS>3</FOLD_THRESH_FINS>
   </COMPONENT_TYPE_GROUP>
  </LIB>
 </COMPONENT_TYPE_GROUPS>
 <MAPPINGS>
  <LIB NAME=“cdn20”>
   <CELL NAME=“nfinFET”>
    <COMPONENT_TYPE>
     <NAME>nfin</NAME>
     <LIB>cdn20</LIB>
    </COMPONENT_TYPE>
   </CELL>
   <CELL NAME=“pfinFET”>
    <COMPONENT_TYPE>
     <NAME>pfin</NAME>
     <LIB>cdn20</LIB>
    </COMPONENT_TYPE>
   </CELL>
  </LIB>
 </MAPPINGS>
</PHYSICAL>

The example above indicates that the FinFET devices have the width parameter set. As a result, the devices will automatically fold when the folding threshold defined by the FOLD_THRESH_FINS is reached such that each fold gets a whole number of fins.

Example 5 LAM File depicting the use of the COMPTYPE_PARAMS attribute

<!DOCTYPE LAM_FILE> 
<LIBRARY_DATA> 
<VERSION NUMBER="0.3" /> 
<PHYSICAL> 
<COMPONENT_TYPE_GROUPS> 
<LIB NAME="cdn20FFBase" > 
<COMPONENT_TYPE_GROUP NAME="NFIN" > 
<COMP_CLASS>NFIN</COMP_CLASS> 
<NUM_FINS>nFins</NUM_FINS> 
<FOLD_THRESH_FINS>2</FOLD_THRESH_FINS> 
<COMPTYPE_PARAMS>list(list("rSADPShoulder" "auto") list("lSADPShoulder" "auto"))</COMPTYPE_PARAMS>
        </COMPONENT_TYPE_GROUP>                                                                             
        <COMPONENT_TYPE_GROUP NAME="NFIN2" >                                                                
          <COMP_CLASS>NFIN</COMP_CLASS>                                                                     
          <NUM_FINS>nFins</NUM_FINS>                                                                        
          <FOLD_THRESH_FINS>2</FOLD_THRESH_FINS>                                                            
          <COMPTYPE_PARAMS>list(list("rSADPShoulder" "au*") list("lSADPShoulder" "*to"))</COMPTYPE_PARAMS>  
        </COMPONENT_TYPE_GROUP>                                                                             
        <COMPONENT_TYPE_GROUP NAME="NFIN3" >                                                                
          <COMP_CLASS>NFIN</COMP_CLASS>                                                                     
          <NUM_FINS>nFins</NUM_FINS>                                                                        
          <FOLD_THRESH_FINS>2</FOLD_THRESH_FINS>                                                            
          <COMPTYPE_PARAMS>list(list("rSADPShoulder" "*au*") list("lSADPShoulder" "*ut*"))</COMPTYPE_PARAMS>
        </COMPONENT_TYPE_GROUP>                                                                             
        <COMPONENT_TYPE_GROUP NAME="PFIN" >                                                                 
          <COMP_CLASS>PFIN</COMP_CLASS>
          <WIDTH>nFins</WIDTH>
          <WIDTH_FIN>3e-06</WIDTH_FIN>
          <FOLD_THRESH_FINS>3</FOLD_THRESH_FINS>
        </COMPONENT_TYPE_GROUP>
      </LIB>
    </COMPONENT_TYPE_GROUPS>
    <MAPPINGS>
      <LIB NAME="cdn20FFBase" >
        <CELL NAME="nfinFET" >
          <COMPONENT_TYPE>

            <LIB>cdn20FFBase</LIB>
          </COMPONENT_TYPE>
        </CELL>
        <CELL NAME="nfinFET" >
          <COMPONENT_TYPE>
            <NAME>NFIN2</NAME>
            <LIB>cdn20FFBase</LIB>
          </COMPONENT_TYPE>
        </CELL>
        <CELL NAME="nfinFET" >
          <COMPONENT_TYPE>
            <NAME>NFIN3</NAME>
            <LIB>cdn20FFBase</LIB>
          </COMPONENT_TYPE>
        </CELL>

          <COMPONENT_TYPE>
            <NAME>PFIN</NAME>
            <LIB>cdn20FFBase</LIB>
          </COMPONENT_TYPE>
        </CELL>
      </LIB>
    </MAPPINGS>
  </PHYSICAL>
</LIBRARY_DATA>

The LAM file defines multiple component types with different parameters and assigns the same cell with multiple component types. This allows creating different component types based on different cell properties for the same cell.

Example 6 LAM File depicting the use of the COMPTYPE_PARAMS attribute, defining some component types and assigning cells.

<!DOCTYPE LAM_FILE>
<LIBRARY_DATA>
  <VERSION NUMBER="0.3" />
  <PHYSICAL>
    <COMPONENT_TYPE_GROUPS>
      <LIB NAME="testLib" >
<COMPONENT_TYPE_GROUP NAME="NDevice1" >
<COMP_CLASS>NFIN</COMP_CLASS>
<NUM_FINS>nFins</NUM_FINS>
<FOLD_THRESH_FINS>2</FOLD_THRESH_FINS>
<COMPTYPE_PARAMS>list(list("param1" value1) list("param2" value2))</COMPTYPE_PARAMS>
</COMPONENT_TYPE_GROUP>
<COMPONENT_TYPE_GROUP NAME="PFIN" >
<COMP_CLASS>PFIN</COMP_CLASS>
<NUM_FINS>nFins</NUM_FINS>
<FOLD_THRESH_FINS>3</FOLD_THRESH_FINS>
<COMPTYPE_PARAMS>list(list("param1" value1) list("param2" value2) list("param3" value3)</COMPTYPE_PARAMS>
</COMPONENT_TYPE_GROUP>
      </LIB>
    </COMPONENT_TYPE_GROUPS>
    <MAPPINGS>
      <LIB NAME="testLib" >
<CELL NAME="nfinCell1" >
<COMPONENT_TYPE>
<NAME>NDevice1</NAME>
<LIB>testLib</LIB>
</COMPONENT_TYPE>
</CELL>
<CELL NAME="pfinCell1" >
<COMPONENT_TYPE>
<NAME>PDevice1</NAME>
<LIB>testLib</LIB>
</COMPONENT_TYPE>
</CELL>
</LIB>
    </MAPPINGS>
  </PHYSICAL>
</LIBRARY_DATA>
