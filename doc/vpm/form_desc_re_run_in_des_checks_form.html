Run In-Design Checks

Runs In-Design checks and generates the log and violations files in the foreground.

Field Name Description

Scope

Specifies whether to run the checks on the complete design or the currently descended hierarchical instance of the top cellview.

Library

Specifies the library of the cellview to be checked.

Cell

Specifies the cell of the cellview to be checked.

View

Specifies the view name of the cellview to be checked.

Instance Path

Specifies the path to the currently descended hierarchical instance on which the checks are to be run. This field appears only when the Hierarchical Instance option is selected for the Scope field.

Log File

Specifies the name of the log file to be generated at the end of checks.

The file path from the last session is retained.

Violations File

Specifies the name of the file that will contain the design violations at the end of checks.

The file path from the last session is retained.
