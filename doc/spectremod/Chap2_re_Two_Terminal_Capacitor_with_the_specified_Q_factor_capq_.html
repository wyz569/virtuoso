Two Terminal Capacitor with the specified Q factor (capq)

Description

The 'capq' instance is a capacitor with the Q factor 'q' specified at a particular frequency 'fq'. The parameter 'mode' specifies the exact form of the frequency dependence of the real part of the admittance Re(Y).

mode=1: Constant real part of the admittance Re(Y)=const.

mode=2: Re(Y) increases proportional to sqrt(freq).

mode=3: Re(Y) increases linearly with frequency.

mode=4: Re(Y) decreases proportional to (freq/fq)^alph.

This device is supported within altergroups.

The capq instance is an RC parallel branch with R being frequency-dependent and C=const.

Instance parameters: c, mode, fq, q.

The equations are written in terms of admittance Y, with w=2*Pi*freq and wq=2*Pi*fq:

mode=1:   Re(Y)=wq*c/q=const; Im(Y)=w*C.

mode=2:   Re(Y)=c*sqrt(wq*w)/q; Im(Y)=w*C.

mode=3:   Re(Y)=w*c/q; Im(Y)=w*C.

mode=4: Re(Y)=w*c/Qf; Im(Y)=w*c; Qf=q*(freq/fq)^alph.

Sample Instance Statement:

Without model:

cq2 (1 0) capq c=2.5u mode=2 fq=1.3G q=120

cq2 (1 0) capq c=2.5u mode=4 fq=1.3G q=120 alph=1

With model:

cq2 (1 0) proc_capq

Sample Model Statement:

model proc_capq capq c=2u mode=1 fq=1.3G q=100

model proc_capq capq c=2u mode=4 fq=1.3G q=100 alph=1

This device does not support behavioral expressions.

Instance Syntax

Name  1  2 ModelName parameter=value ...

Instance Parameters

c

(F)

Capacitance.

ic

(V)

Initial condition(alias=lv2).

m

1

Multiplicity factor.

trise

(C)

Temperature rise from ambient.

q

50.0

Quality factor measured at the frequency 'fq'.

fq

1.0e8 Hz

Frequency at which 'q' is measured.

mode

1

Integer parameter that selects the frequency dependence.

isnoisy

yes

Should capq resistance generate noise.Possible values are no and yes.

alph

0

Scaling factor for Q in 'capq', qf=q*(freq/fq)^alph, Y=2*pi*freq*c*(1/qf+j).

The instance parameter mode defaults to 1, if not specified.

The Q factor defaults to q=50, and fq defaults to fq=0.1 GHz.

In transient simulations, the DC values are used.
