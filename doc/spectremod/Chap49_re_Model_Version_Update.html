Spectre Circuit Simulator Components and Device Models Reference
Product Version 23.1, June 2023

Model Version Update

Version 102.94

  • Sign of S/D resistance power in self heating model was corrected.

Version 102.92

  • Some bugs have been fixed.
  • Following enhancements have been made in this release:
    • Self-heating temperature is clamped to a maximum device temperature.
    • Added binning equations to QME parameters.
    • Implemented Drain-Induced-Vth Shift (DITS) module.

Version 102.90

  • Added Vbg dependence in the gate current module.
  • Added external thermal node, t.
  • Added descriptions for the binning parameters.
  • The ifdef statements are removed.
  • Typo corrected in the technical manual and Bin term default values.
  • Improved gds fitting in low gate overdrive.
  • Added numerical protection for sqrt(phist) term
  • Alignment of contribution (<+) lines is fixed.
  • The pattern for no unit expression is now used for the binning parameters also.
  • $strobe for EF and NTNOI are removed.
  • gth and cth in the else case are initialized.
  • CONSTCtoK is replaced with P_CELSIUS0.
  • The strobe commands used for diagnostic purpose are removed.

Version 102.80

  • Expressions of output variables CGG, CSG, and CDG have been fixed.
  • Back-gate dependence of GIDL/GISL currents has been added
  • Some additional binning parameters have been made binnable
  • Range of several parameters has been adjusted
  • NQSMOD (intrinsic-input impedance) has been added similar to BSIM-CMG. Two binnable model parameters XRCRG1 and XRCRG2 used to adjust the bias-dependent intrinsic-input resistance have also been added.
  • First order NR solution in surface potential calculation has been removed to avoid convergence issues
  • Output variables t_total_k, t_total_c, and t_delta_sh have been added for self-heating temperature

Version 102.70

  • Some bugs have been fixed.
  • Following enhancements have been made in this release:
    • A new smoothing function smoothminx has been introduced to clamp the minimum value of dmob/dmobs/dvsat at 1 (no degradation) with deltax being the smoothing factor.
    • Two new model parameters DMOBCLAMP and DVSATCLAMP with default values of 0.01have been added.
    • A new operating point parameter VTH has been added.

Version 102.61

Following enhacements have been made in this release:

  • The description of chargemod parameter has been changed
  • Minimum clamp has been updated in rgate for better convergence
  • New operating point parameters have been added
  • RDSMOD =2 has been introduced to enable source/drain node collapse
