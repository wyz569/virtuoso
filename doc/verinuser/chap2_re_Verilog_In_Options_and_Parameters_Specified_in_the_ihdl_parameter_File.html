Verilog In Options and Parameters Specified in the ihdl_parameter File

The following table describes the parameters defined in ihdl_parameter file and their corresponding form fields.:

Parameters

Description

Import Options

These parameters correspond to option fields on the Import Options tab.

dest_sch_lib

Specifies the target library.

Form Field: Target Library Name

dest_sch_lib := <target library name>

ref_lib_list

Specifies thForm fielde reference libraries. Use a comma (,) to separate library names.

Form Field: Reference Libraries

ref_lib_list := basic, sample, US_8ths

hdl_view_name

Specifies the view in the pre-compiled library which is used to find the IR for the cell while using pre-compiled libraries.

Form Field: HDL View Name

hdl_view_name := hdl

ignore_node_file

Specifies the text file that contains the list of modules that you do not want to import.

Form Field: Ignore Modules File 

ignore_node_file := <file path>
This version of Verilog In does not support stop modules. Therefore, the stop_node_file parameter is no longer available.

import_mod_file

Specifies the text file, which lists the modules that you need to import. The default value of this parameter is null.

Form Field: Import Modules File

import_mod_file := <file path>

structural_views

Specifies which views to create for structural modules.

Form Field: Structural Modules

Valid values are:

  • schematic only
  • netlist only
  • functional only
  • schematic and functional
  • netlist and functional

structural_views:= 4

OA_structural_view_names

Specifies the views to be created in the target library. The default values are functional, netlist, schematic, and symbol respectively.

Form Field: Import Modules As

functional_view_name := functional
netlist_view_name := netlist
schematic_view_name := schematic
symbol_view_name := symbol

log_file_name

Specifies the file that lists all error messages and log messages.

Form Field: Log File 

log_file_name := <file path>

work_area

Specifies the directory where Verilog In stores internal data. This parameter is useful when importing large designs and space in /tmp is limited. The default is /tmp.

Form Field:Work Area

work_area := <directory name>

map_file_name

Specifies the file that lists original escaped names and corresponding mapped names.

Form Field: Name Map Table 

map_file_name := <file path>

import_if_exists

Specifies whether to import a module that has the same name as a module that already exists in the target library. When you set this value to 1, the module is imported. When you set this value to its default 0, the module is not imported.

Form Field: Overwrite Existing Views 

import_if_exists := 0
The overwrite_option works in the same way as the import_if_exists option.

overwrite_symbol

Specifies whether to overwrite symbol of a module that already exists in the target library. By default, this parameter is set to 0 and symbols are not overwritten. Set this value to the following values:

  • 1 to overwrite symbols created by Verilog In (symbols have the createdBy property set as VerilogIn).
  • 2 to overwrite symbols created by Text-to-Symbol generator or any other tool (symbols have the createdBy property set as any value other than VerilogIn).
  • 3 to overwrite all symbols.

Form Field: Overwrite Symbol Views

overwrite_symbol := 1

import_cells

Specifies whether to import a Verilog HDL cell and the view type. When you set this value to its default 0, the cell is imported as a symbol. When you set this value to 1, the cell is imported according to the value of the import_lib_cells option.

Form Field: Verilog Cell Modules 

import_cells := 0

import_lib_cells

When the import_cells option is set to 1, this option specifies a view type other than symbol for the import_cells option

When this option is set to its default 0, the cell is imported as a functional view. When set to 1, the cell is imported according to the value of the structural_views option.

Form Field: Verilog Cell Modules 

import_lib_cells := 0

Global Net Options

These parameters correspond to option fields on the Global Net Options tab.

power_net

Specifies the name of the global power signal in the Verilog design. You can specify only one power net name. All modules in the design recognize this name.

Form Field: Power Net Name

power_net := <user-defined power net name>

If the "!" character exists in the power net name, Verilog In does not add an extra "!" character.

ground_net

Specifies the name of the global ground signal in the Verilog design. You can specify only one ground net name. All modules in the design recognize this name.

Form Field: Ground Net Name

ground_net := <user-defined ground net name>

If the "!" character exists in the ground net name, Verilog In does not add an extra "!" character.

glob_sig_names

Specifies that the nets in the Verilog design file must be treated as global.

Form Field: Global Signals

glob_sig_names := <user-defined global net name>

enable_explicit_port_checking

If the module description contains explicitly defined empty ports:

  • When the flag is set, a functional view is created.
  • When the flag is not set, a symbol with a null port is created.

If the instance line contains undefined ports in case of explicit connections:

  • When the flag is set, a functional view is created.
  • When the flag is not set, a schematic is created.
enable_explicit_port_checking := 1

To enable this functionality when the import is done through GUI, you can set the enableExplicitPortChecking environment variable as follows:

In CIW:

envSetVal("ihdl" "enableExplicitPortChecking" 'boolean t)

In the .cdsenv file:

"ihdl" "enableExplicitPortChecking" 'boolean nil

create_ifc_func_view

Imports only the port information in any functional views that are created. All instance information is ignored. 

create_ifc_func_view := 1

To enable this functionality when the import is done through CIW, you can set the createIfcFunView environment variable as follows:

In CIW:

envSetVal("ihdl" "createIfcFunView" 'boolean t)

In the .cdsenv file:

"ihdl" "createIfcFunView" 'boolean nil

Schematic Generation Options

These parameters correspond to option fields on the Schematic Generation Options tab.

sheet_symbol

Specifies which sheet border size Verilog In applies when creating a multisheet schematic. The sheet borders reside in the US_8ths library. When this value is none, Verilog In creates an infinite sheet schematic.

Form Field: Sheet Symbol

sheet_symbol := Asize

page_row_limit

Specifies the maximum number of rows on each sheet. Usually, the maximum number of rows is expressed as a number between 1 and 1024. If missing, the limit is automatically set to 1024.

Form Field: Maximum Number of Rows

page_row_limit := 512

page_col_limit

Specifies the maximum number of columns on each sheet. Usually the maximum number of columns is expressed as a number between 1 and 1024. If missing, the limit is automatically set to 1024.

Form Field: Maximum Number of Columns

page_col_limit := 1024

label_height

Specifies the size of the font used for pin, wire, and instance labels. The value must be an integer greater than or equal to 1.

Form Field: Font Height

label_height := 10

line_line_spacing

Specifies the spacing in inches between nets flowing in a channel. The value must be a real number in the range of 0.125 to 0.625.

Form Field: Line to Line Spacing

line_line_spacing := 0.2

line_component_spacing

Specifies the spacing in inches between a component and the nearest net flowing in a channel. The value must be a real number in the range of 0.125 to 0.625.

Form Field: Line to Component Spacing

line_component_spacing := 0.2

density_level

Specifies the density of the schematic. The input must be an integer from 0 to 100, where 100 is the most dense and 0 is the least dense.

Form Field: Component Density

density_level := 0

cds_thru_symbol

Specifies the library, cell, and view names for the cds_thru symbol. The default is basic cds_thru symbol.

Form Field: Through CellView For Shorted Ports

cds_thru_symbol := basic cds_thru symbol

cont_assign_Symbol

Specifies the library, cell, and view names for the patch symbol. The default is basic patch symbol.

Form Field: Continuous assignment symbol

cont_assign_symbol := basic patch symbol

pin_placement

Specifies the IO pin direction assignment for the schematic view.

Form Field: Pin Placement

pin_placement := all_sides
pin_placement := left_and_right_sides
pin_placement := file, <pin_placement_file_name>

create_snap_space_prop

Specifies whether the creation of snap space properties must be enabled or disabled. Snap space can be defined as the minimum distance the cursor moves, in user units. The default distance (0.0625) is equal to half the default grid spacing distance. When this option is set to 0, then the x-y Snap Space properties are not created on the cellview.

create_snap_space_prop := 0

ref_sch_list

Specifies comma-separated schematic or symbol views that the tool can look up for missing terminals on an instance line in the input Verilog file.

If an instance line in the Verilog file has more number of terminals than the number of terminals on the identified instance symbol master, Verilog In uses the specified schematic or symbol views to look for the required additional terminals. The reference schematic or symbol views can contain terminals or nets with net expressions whose names match the names of the additional terminals on the instance line. The tool extracts the netExpression property name from those inherited terminals or nets. It then appropriately sets the netSet property on the instance for the missing terminals. The netSet property name is set using the netExpression property name obtained from the referenced schematic. The netSet property value is set from the net connected to the instance line in the Verilog file.

Form Field: Reference Schematic/Symbol Views For Inherited Connections

ref_sch_list := schematic, sch

pnr_max_inst

Specifies the maximum number of instances that the design must have to trigger the generation of a place-only schematic.

Form Field: Instances Less Than

pnr_max_inst := 20000

pnr_max_port

Specifies the maximum number of ports that the design must have to trigger the generation of a place-only schematic.

Form Field: Ports Less Than

pnr_max_port := 5000
